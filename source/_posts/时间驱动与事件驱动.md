实际上是数据结构的内容

# 1.概念
## 离散事件模拟
#### 基本概念
- 系统状态在**离散的时间点**发生变化
- 状态变化由事件触发
- 事件之间的时间间隔不固定
## 时间驱动模拟
#### 基本概念
- 按固定时间步长推进时钟
- 在每个时间点检查所有可能发生的事件
- 类似动画的逐帧播放
#### 特点
- 时间精度：由时间步长决定（如1秒、0.1秒）
- 实现简单：直观易懂，逻辑直接
- 计算开销：每一步都执行，可能很多"空检查"
- 适用场景：状态连续变化、事件密集、简单系统
#### 伪代码示例
```
for (time = 0; time < endTime; time += timeStep) {
    // 每秒/每0.1秒检查：
    if (有顾客到达？) { 处理到达(); }
    if (服务完成？) { 处理离开(); }
    if (其他条件？) { 处理其他(); }
    
    更新统计();
}
```

## 事件驱动模拟
#### 基本概念
- 跳跃到下一个事件时间点
- 只在实际事件发生时处理

#### 基本结构
生产者(producer) - 事件代理(event broker) - 消费者(subsriber)结构
1. 生产者生产事件
2. 将事件推送到事件代理
3. 由消费者处理事件

#### 特点
- 时间精度：任意精度（浮点数时间）
- 计算效率：只处理实际事件，跳过空闲时间
- 适用场景：状态离散变化、事件稀疏、复杂系统
#### 伪代码示例
```
while (!eventQueue.empty()) {
    // 取出最早的事件
    Event e = eventQueue.pop();
    currentTime = e.time;  // 直接跳到事件时间
    
    // 处理该事件
    switch (e.type) {
        case ARRIVAL: 处理到达(); 生成下一个到达事件(); break;
        case DEPARTURE: 处理离开(); 检查队列(); break;
    }
}
```
